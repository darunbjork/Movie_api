MyFlix API Documentation

Endpoints

1. Return a list of ALL movies to the user

URL: /movies

Method: GET

Description: Returns a JSON object containing a list of all movies.

Request Data: None

Response Data: A JSON array of movie objects.

GET /movies HTTP/1.1
Host: localhost:8080

Response:
[
  {
    "id": 1,
    "title": "Inception",
    "genre": "Sci-Fi",
    "director": "Christopher Nolan",
    "description": "A mind-bending thriller",
    "imageUrl": "",
    "featured": true
  },
  {
    "id": 2,
    "title": "The Dark Knight",
    "genre": "Action",
    "director": "Christopher Nolan",
    "description": "Batman fights Joker",
    "imageUrl": "",
    "featured": true
  }
]

2. Return data about a single movie by title to the user

URL: /movies/:title

Method: GET

Description: Returns data about a single movie specified by title.

Request Data: :title (parameter in URL)

Response Data: A JSON object containing data about the specified movie.

GET /movies/Inception HTTP/1.1
Host: localhost:8080

Response:
{
  "id": 1,
  "title": "Inception",
  "genre": "Sci-Fi",
  "director": "Christopher Nolan",
  "description": "A mind-bending thriller",
  "imageUrl": "",
  "featured": true
}

3. Return data about a genre by name/title

URL: /genres/:name

Method: GET

Description: Returns data about a genre specified by name.

Request Data: :name (parameter in URL)

Response Data: A JSON object containing data about the specified genre.

GET /genres/Sci-Fi HTTP/1.1
Host: localhost:8080

Response:
{
  "name": "Sci-Fi",
  "description": "Science fiction is a genre that uses speculative, fictional science-based depictions of phenomena."
}

4. Return data about a director by name

URL: /directors/:name

Method: GET

Description: Returns data about a director specified by name.

Request Data: :name (parameter in URL)

Response Data: A JSON object containing data about the specified director.

GET /directors/Christopher%20Nolan HTTP/1.1
Host: localhost:8080

Response:
{
  "name": "Christopher Nolan",
  "bio": "Christopher Edward Nolan is a British-American film director, producer, and screenwriter.",
  "birthYear": 1970,
  "deathYear": null
}

5. Allow new users to register

URL: /users

Method: POST

Description: Registers a new user.

Request Data: A JSON object containing user details.

POST /users HTTP/1.1
Host: localhost:8080
Content-Type: application/json

{
  "username": "john_doe",
  "password": "password123",
  "email": "john@example.com",
  "birthday": "1990-01-01"
}

Response:
{
  "username": "john_doe",
  "email": "john@example.com",
  "birthday": "1990-01-01"
}

6. Allow users to update their user info (username)

URL: /users/:username

Method: PUT

Description: Updates a user's information specified by username.

Request Data: A JSON object containing updated user details.

PUT /users/john_doe HTTP/1.1
Host: localhost:8080
Content-Type: application/json

{
  "password": "newpassword123",
  "email": "newjohn@example.com",
  "birthday": "1991-01-01"
}

Response:
{
  "username": "john_doe",
  "email": "newjohn@example.com",
  "birthday": "1991-01-01"
}

7. Allow users to add a movie to their list of favorites

URL: /users/:username/movies/:movieID

Method: POST

Description: Adds a movie to a user's list of favorite movies.

Request Data: None

Response Data: A message indicating that the movie has been added to favorites.

POST /users/john_doe/movies/1 HTTP/1.1
Host: localhost:8080

Response:
Movie with ID 1 was added to user john_doe's favorite list.

8. Allow users to remove a movie from their list of favorites

URL: /users/:username/movies/:movieID

Method: DELETE

Description: Removes a movie from a user's list of favorite movies.

Request Data: None

Response Data: A message indicating that the movie has been removed from favorites.

DELETE /users/john_doe/movies/1 HTTP/1.1
Host: localhost:8080

Response:
Movie with ID 1 was removed from user john_doe's favorite list.

9. Allow existing users to deregister

URL: /users/:username

Method: DELETE

Description: Deregisters an existing user specified by username.

Request Data: None

Response Data: A message indicating that the user has been deregistered.

DELETE /users/john_doe HTTP/1.1
Host: localhost:8080

Response:
User john_doe was deregistered.
